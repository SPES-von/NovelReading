# 基于 Vue 与 Spring Boot 的轻小说阅读系统设计与实现

---

## 摘要

随着移动互联网与数字阅读的普及，轻小说作为一种流行的文学体裁受到越来越多读者的喜爱。传统在线阅读平台往往功能单一、体验欠佳，难以满足读者对个性化阅读与多端一致体验的需求。本文设计并实现了一个基于 Spring Boot 框架的轻小说阅读系统，采用 Vue 3 前端与 Spring Boot 后端相结合的前后端分离架构，使用 MySQL 存储业务数据、Redis 进行缓存、JWT 进行身份认证，并集成百度语音合成 API 实现听书功能。系统按角色划分功能：**读者端**支持用户注册与登录、小说浏览与分类筛选、全文搜索与多维度排行榜、小说详情与在线阅读、个人书架管理、阅读进度同步、个人资料与阅读偏好管理以及章节语音朗读；**管理端**支持用户管理、小说管理（上架与下架、新增编辑、章节维护）、分类管理、评论管理、数据统计与系统设置。系统在读者端提供首页多模块推荐（特色推荐、强推榜、新书榜、热点榜、完本推荐、热门小说等）、轮播与榜单展示等交互模块，有效改善了传统平台效率低、体验差的问题，提升了阅读体验与用户粘性。经功能测试与部署验证，系统运行稳定。本系统验证了 Vue 3 与 Spring Boot 在数字阅读领域开发中的效率与实用性，为轻小说垂直领域提供了一种可行的解决方案。

**关键词**：轻小说阅读系统；Vue 3；Spring Boot；前后端分离；JWT；Redis 缓存

## Abstract

With the popularity of mobile Internet and digital reading, light novels have become a popular literary genre among readers. Traditional online reading platforms often have single functions and poor experience, which can hardly meet readers' needs for personalized reading and consistent experience across devices. This paper designs and implements a light novel reading system based on the Spring Boot framework, adopting a front-end and back-end separated architecture with a Vue 3 front-end and a Spring Boot back-end, using MySQL for business data storage, Redis for caching, JWT for authentication, and Baidu Speech API for text-to-speech. The system provides role-based functionality: the **reader side** supports user registration and login, novel browsing and category filtering, full-text search and multi-dimensional rankings, novel details and online reading, personal bookshelf management, reading progress synchronization, profile and reading preference management, and chapter narration; the **admin side** supports user management, novel management (review, listing/delisting, chapter maintenance), category management, comment management, data statistics, and system settings. The reader side offers interactive modules such as multi-module home page recommendations (featured, strong recommendation chart, new book chart, hot chart, completed recommendations, hot novels, etc.) and carousel/ranking displays, effectively addressing the low efficiency and poor experience of traditional platforms and improving reading experience and user stickiness. After functional testing and deployment verification, the system runs stably. This system demonstrates the efficiency and practicality of Vue 3 and Spring Boot in digital reading development and provides a feasible solution for the light novel vertical market.

**Keywords**: Light Novel Reading System; Vue 3; Spring Boot; Front-end and Back-end Separation; JWT; Redis Cache

## 第 1 章 绪论

### 1.1 研究背景与意义

近年来，数字阅读产业快速发展，轻小说作为源自日本、融合插画与剧情的文学形式，在国内外拥有大量读者。据相关行业报告，国内轻小说与二次元阅读用户规模持续增长，读者对在线阅读平台的需求已从“能看”转向“好看、好用、个性化”，包括多端一致体验、阅读进度同步、个性化设置、听书等增值功能。传统单体架构或简单静态站点往往将页面渲染与业务逻辑耦合在一起，难以在性能、可维护性和功能扩展性之间取得平衡，也无法很好地支持多端复用与敏捷迭代。

本课题设计并实现一个轻小说阅读系统，采用前后端分离架构，将展示层与业务层解耦，前端专注于用户界面与交互，后端专注于业务逻辑与数据持久化，便于团队分工协作与后续功能扩展。系统整合小说推荐、分类筛选、用户中心、阅读进度与听书等功能，为轻小说爱好者提供一站式的阅读与听书体验。从实践角度，本系统综合运用了当前主流的 Web 开发技术栈，对类似数字阅读平台的设计与实现具有一定的参考价值与借鉴意义。

### 1.2 国内外研究现状

#### 1.2.1 国内研究现状

我国数字阅读与网络文学平台的研发与实践自 21 世纪初起步，至 2010 年代已形成较为成熟的产业格局。据中国音像与数字出版协会发布的《2022 年度中国数字阅读报告》，截至 2022 年，我国数字阅读用户规模达 5.3 亿，同比增长 4.75%，数字阅读产业总体规模达 463.52 亿元，其中大众阅读、有声阅读等板块持续增长[11]。起点中文网自 2002 年创立、2003 年推出 VIP 付费阅读制度以来，与晋江文学城、番茄小说等平台共同构成了国内网络文学与轻小说内容生态的重要支柱；上述平台在移动端与 Web 端均提供书架、阅读进度同步、听书、夜间模式等能力，依托完善的版权体系、丰富的内容供给与便捷的用户体验占据较大市场份额[12]。

学界与产业界在数字阅读领域的研究多集中于**平台架构设计**、**用户体验优化**与**推荐与个性化系统**。例如，阅文集团等企业通过前后端分离架构与 RESTful API 支撑多端一致阅读与听书功能，部分平台引入协同过滤、标签匹配等推荐算法以提升用户留存与付费转化；有声阅读与语音合成技术的接入进一步拓展了阅读场景。相关研究旨在提升内容分发效率、优化多端阅读体验并降低运营成本。当前，国内平台正持续通过智能化系统优化内容推荐、阅读偏好与用户服务，轻小说垂直领域亦有哔哩哔哩漫画等平台涉足，但在开源或可复用的轻小说阅读系统设计与完整实现方面仍相对有限，多数为商业闭源产品，本系统在技术选型上参考上述主流实践，采用 Vue 3 与 Spring Boot 实现轻量而完整的阅读与听书功能，兼顾开发效率与系统可扩展性。

#### 1.2.2 国外研究现状

国外数字阅读与电子书平台的研究与实践可追溯至 21 世纪初。2007 年 11 月，亚马逊推出首款 Kindle 电子书阅读器，采用 E Ink 电子纸显示技术及无线下载与 DRM 版权管理，推动了电子书阅读器与数字阅读市场的快速发展[13]。此后，以 Kindle 商店、苹果 iBooks、谷歌 Play 图书等为代表的平台依托成熟的云同步与多设备阅读能力，形成了“设备—内容—账户”一体化的阅读生态。

在轻小说与漫画等垂直领域，角川集团旗下 BookWalker 自 2014 年起面向海外推出英文版数字漫画与轻小说服务，其技术架构采用前后端分离与云部署（如基于 Next.js 的前端、Rails 后端与 AWS 基础设施），注重多端一致体验与版权保护；部分平台还引入 LCP（Lightweight Content Protection）等 DRM 方案以兼顾内容安全与跨应用阅读[14]。日本集英社的 Shonen Jump+ 等应用则通过移动端优先、章节更新与社区互动等功能强化用户粘性。总体而言，国外研究与产品实践在**平台技术架构**、**个性化推荐与智能分发**以及**跨设备同步与无障碍阅读**等方面积累了较多经验，技术实现上普遍采用 RESTful 或 GraphQL 接口、JWT 或 OAuth 认证及 Redis 等缓存手段，并结合推荐算法与个性化设置提升用户留存。本系统在架构与认证、缓存等选型上参考上述国际主流实践，以适应轻小说阅读场景的需求。

## 第 2 章 相关技术介绍

### 2.1 前端技术

#### 2.1.1 Vue 3

Vue.js 是一套用于构建用户界面的渐进式 JavaScript 框架，由尤雨溪创建并维护；Vue 3 为其第三个主要版本，使用 Proxy 实现响应式、引入 Composition API，适合构建中大型单页应用（SPA）。相较 React、Angular，Vue.js 采用模板语法与单文件组件（.vue），学习曲线平缓且与 Vue Router、Pinia 等官方生态配合紧密，适合多页面、多模块的阅读类 Web 应用。本系统前端使用 Vue 3 组织页面与组件，通过响应式数据驱动视图更新，各功能模块以单文件组件形式组织，便于复用与维护。

#### 2.1.2 Vite

Vite 是新一代前端构建工具，基于原生 ESM 在开发阶段提供极快的冷启动与热更新（HMR），生产构建使用 Rollup 打包、体积小且树摇效果好。相较 Webpack 的全量打包，Vite 按需编译当前模块，项目规模增大时启动与 HMR 仍较快；Vue.js 官方将其作为新项目推荐构建工具。本系统使用 Vite 作为 Vue.js 项目的构建与开发服务器，以提升开发效率。

#### 2.1.3 Vue Router 与 Axios

Vue Router 是 Vue.js 官方路由管理器，支持嵌套路由、路由守卫与 HTML5 History 模式；Axios 是基于 Promise 的 HTTP 客户端，支持拦截器与取消请求，常用于 RESTful 调用。SPA 需由前端管理路由与 URL，Vue Router 与 Vue.js 深度集成，便于实现登录校验与懒加载；Axios 相较原生 fetch 更易统一附加 JWT、处理 401。本系统使用 Vue Router 的 createWebHistory 配置首页、登录、书架、阅读页等路由，对 Axios 封装后在请求拦截器中附加 JWT、在响应拦截器中统一处理 401，便于业务层专注逻辑。

### 2.2 后端技术

#### 2.2.1 Spring Boot

Spring Boot 基于 Spring 框架，提供自动配置、内嵌服务器与起步依赖，可快速搭建生产级 Java 应用。相较传统 Spring 需大量 XML 与配置，Spring Boot 约定优于配置、开箱即用，适合在有限周期内完成后端开发；与 Node.js、Python 等相比，Java 生态在企业级 Web 与 RESTful API 方面成熟稳定、类型安全与可维护性较好。本系统后端采用 Spring Boot，整合 Spring Web（RESTful API）、Spring Data JPA、Spring Security、Spring Data Redis，实现业务逻辑与接口的统一管理。

#### 2.2.2 Spring Data JPA 与 MySQL

JPA（Java Persistence API）是 Java 持久化规范，Spring Data JPA 在其上提供仓库抽象与约定式查询，可大幅减少手写 SQL 与样板代码，相较 MyBatis 更侧重“对象—表”映射与快速 CRUD，适合业务实体较多的系统。MySQL 是成熟的开源关系型数据库，支持事务与索引、生态与运维资料丰富，字符集 utf8mb4 可正确存储中文及多语言。本系统使用 JPA 实体映射 MySQL 表结构，通过 Repository 接口完成小说、章节、用户、书架、阅读进度等数据的增删改查，数据库字符集采用 utf8mb4、排序规则为 utf8mb4_unicode_ci。

#### 2.2.3 Spring Security 与 JWT

Spring Security 是 Spring 生态中的安全框架，支持认证与授权；JWT（JSON Web Token）是一种无状态令牌，服务端签发后由客户端在请求头中携带，无需服务端保存会话，便于水平扩展。相较基于 Session 的有状态认证，JWT 适合前后端分离与多端接入；相较引入 OAuth2 等完整协议，JWT 更轻量、适合本系统“登录即签发、接口校验”的需求。本系统采用 Spring Security 整合 JWT：登录成功后签发 JWT，受保护接口（书架、个人资料、阅读偏好）校验 Token 后放行，公开接口（小说列表、章节、登录注册）无需 Token；密码使用 BCrypt 加密存储。

#### 2.2.4 Redis

Redis 是高性能内存键值存储，常用于缓存、会话与计数器。相较应用内本地缓存，Redis 可多实例共享、支持过期与多种数据结构，适合缓存热点数据以降低数据库压力；相较 Memcached，Redis 支持更丰富的数据类型与持久化选项。本系统使用 Spring Data Redis 对热门小说数据与用户阅读进度进行缓存，在应用层配置过期与更新策略，以提升接口响应速度并保证数据一致性。

### 2.3 其他技术

#### 2.3.1 百度语音合成 API

百度开放平台提供语音合成（TTS）接口，可将文本按指定音色、语速等参数转换为语音音频流，支持中文及多语种。相较预先录制真人朗读，语音合成无需配音人力、内容更新时无需重录，可随章节内容实时生成；相较部分开源 TTS 引擎，百度接口在中文自然度与稳定性方面表现较好，且以 HTTP API 形式提供、便于后端集成。本系统使用该接口实现听书功能。

#### 2.3.2 前后端分离与跨域

系统采用前后端分离以实现界面与业务解耦，便于分工协作与独立部署：前端为 Vue.js（Vue 3）单页应用，后端为 Spring Boot 提供 RESTful API，通过 HTTP/HTTPS 交互并使用 JWT 进行无状态认证。开发阶段可在前端构建工具中配置代理（如将 `/api` 转发到后端服务）以避免浏览器跨域限制，部署阶段可通过 Nginx 反向代理或后端 CORS 配置允许指定来源访问；该方式既能提升开发迭代效率，也有利于后续扩展管理端与多端接入。

## 第 3 章 需求分析

### 3.1 系统可行性分析

#### 3.1.1 技术可行性

本系统采用前后端分离架构，前端使用 Vue.js（Vue 3）与 Vite 构建单页应用，后端使用 Spring Boot 提供 RESTful API，数据层采用 MySQL 并结合 Redis 缓存，认证采用 JWT。上述技术均为当前 Web 开发中的主流成熟方案，社区活跃、文档完善、学习与排错成本可控，能够满足本系统在小说检索、章节阅读、书架管理、阅读进度同步与听书等场景下对性能、稳定性与可维护性的要求。Spring Boot 具备自动配置和模块化整合能力，便于快速搭建后端服务并支撑接口扩展；Vue.js 组件化开发与 Vite 快速构建机制有利于提升前端开发效率与页面交互体验；MySQL 在结构化业务数据管理方面稳定可靠，Redis 可降低热点读请求对数据库的压力。因此，从技术成熟度、实现难度与后续扩展性来看，本系统具备良好的技术可行性。

#### 3.1.2 经济可行性

本系统在开发与部署阶段以开源技术为主，整体成本较低。开发阶段所使用的 Spring Boot、Spring Security、Spring Data JPA、Vue.js、Vite、Vue Router、Axios、MySQL、Redis 均可免费使用；开发工具采用 IntelliJ IDEA 社区版（免费）与 Cursor（付费约 280 元），除 Cursor 订阅外无需采购其他商业软件授权，工具投入在可接受范围内。部署阶段可先在本地环境完成联调与测试，再按需要部署到轻量云主机，服务器系统与 Nginx 等基础软件同样可采用开源方案。由于系统主要面向 Web 端，硬件要求适中，日常运维成本主要集中在基础云资源与带宽开销，处于学生项目或中小规模应用可承受范围内。综合评估，项目在开发投入、部署费用与后期维护方面均具备较好的经济可行性。

#### 3.1.3 操作可行性

本系统采用 B/S 架构，用户通过浏览器即可访问，无需额外安装客户端，操作门槛较低。读者端界面以阅读流程为主线，提供首页推荐、分类筛选、搜索、阅读、书架、进度同步与听书等常用功能，页面导航清晰、交互路径简洁，能够满足日常阅读与个性化设置需求；管理端提供用户管理、小说管理、分类管理、评论管理、数据统计与系统设置等功能，管理员可通过统一后台完成内容审核与运营维护。整体来看，系统前后台功能完整、使用方式直观，具备良好的操作可行性。

### 3.2 系统业务分析

本系统功能需求主要以读者为中心，覆盖从浏览发现到阅读、收藏与听书的完整流程。读者端主要功能有注册登录、首页推荐与多模块展示、小说浏览与分类筛选、小说详情与在线阅读、全文搜索与排行榜、个人书架管理、个人资料与阅读偏好管理、阅读进度同步以及听书（语音合成）等；访客可浏览与阅读，注册用户还可使用书架、个人中心与听书等需登录功能。对于管理员而言，系统提供用户管理、小说管理（上架与下架、新增编辑、章节维护）、分类管理、评论管理、数据统计与系统设置等功能，便于对用户、小说、分类、评论及系统参数进行统一管理与运营维护。

本系统也有非功能需求，主要涉及性能、安全性、可用性与兼容性等，重点考虑系统是否能在现代浏览器下稳定运行、通过缓存与接口优化控制响应时间，是否通过密码加密与 JWT 鉴权保障用户数据与接口安全，以及阅读页是否支持字体与主题等个性化设置以提升可用性，上述分析为后续需求细化与设计提供依据。

### 3.3 系统功能需求分析

#### 3.3.1 系统参与者与系统功能需求描述

本系统的参与者包括三类：**访客**（未登录用户）、**注册用户**与**系统管理员**，分别对应读者端与管理端的使用场景，三类角色及其对应功能均已实现。

**访客**无需登录即可使用读者端部分功能：浏览首页多模块推荐（特色推荐、强推榜、新书榜、热点榜、完本推荐、热门小说等）、按文库与标签等条件筛选小说、查看小说详情与在线阅读、使用全文搜索与排行榜、对小说送鲜花等；访客不能使用个人书架、个人中心、阅读进度同步与听书等需登录后使用的功能。**注册用户**在访客能力基础上，可完成注册与登录，使用个人书架（加入或移出书架）、个人中心（查看与修改资料、阅读偏好）、阅读进度同步以及听书（章节语音合成）等功能，从而形成从发现、阅读到收藏与续读的完整使用流程。**系统管理员**通过管理端独立入口登录，负责后台运营与内容管理，具备以下职责：用户管理（管理用户账号信息、处理用户反馈与违规账号）、小说管理（上架与下架、新增编辑、章节维护）、分类管理（维护文库与标签等分类体系）、评论管理（审核与删除违规评论）、数据统计（用户阅读数据与小说热度等运营报表）、系统设置（配置系统参数、维护服务器状态与日志）；管理端与读者端在入口与权限上隔离，仅管理员可访问管理功能。

上述参与者与功能对应读者端首页、小说、阅读、用户、书架、听书等模块以及管理端六大管理模块，系统已按上述需求完成开发并投入使用。

### 4.1 系统总体架构设计

本系统按角色划分职责，与典型 Web 应用中的多端分工相对应：**访客**与**注册用户**通过读者端完成内容发现与阅读闭环；**系统管理员**通过管理端完成用户、小说、分类、评论及运营数据的维护与配置。访客可在不登录情况下浏览首页推荐、检索与在线阅读等公开能力；注册用户在此基础上使用书架、个人中心、阅读进度与听书等需鉴权功能；管理员独立使用后台模块，与读者端在入口与权限上隔离。为便于部署维护、降低终端要求并支持后续扩展，系统采用浏览器/服务器（B/S）模式，并以架构图概括各层职责与数据流向，如图 4-1 所示。

#### 4.1.1 B/S 架构与前后端分离

系统采用 **B/S 架构**与**前后端分离**：浏览器仅负责界面展示与交互，业务规则与数据持久化集中在服务端，部署与升级主要发生在服务器侧，客户端无需安装专用程序，有利于控制运维成本并适配 PC 与移动浏览器等多种终端。前后端通过统一的 **RESTful API** 交互，便于多端复用同一套接口、前后端并行开发与独立部署。读者端为 **Vue 3** 单页应用（SPA），承载首页、小说浏览、在线阅读、书架与个人中心等页面；管理端为管理后台（可为同一 Vue 工程下的 `/admin` 路由或独立部署），承载用户管理、小说管理、分类管理、评论管理、数据统计与系统设置等页面。两套前端均经 **HTTP/HTTPS** 访问同一 **Spring Boot** 后端，后端结合路径前缀（如 `/api` 与 `/api/admin`）及 **JWT** 载荷中的角色声明区分读者端与管理端请求，从而在单一服务内实现权限边界清晰的 **RBAC** 式访问控制。

#### 4.1.2 Spring Boot 服务端分层架构

服务端基于 **Spring Boot** 构建，内部采用与 **MVC** 思想一致的分层结构，各层职责边界如下。**控制层（Controller）** 负责接收 HTTP 请求、参数校验与响应封装，将 RESTful 资源映射为具体接口，不承载复杂业务规则。**业务逻辑层（Service）** 实现小说检索、书架、阅读进度、评论审核、统计聚合等应用场景下的流程与规则，并可协调缓存与外部调用。**数据访问层** 由 **Spring Data JPA** 的 **Repository** 接口实现，完成实体与关系型存储之间的映射及基本 CRUD 与查询；**实体类（Entity）** 与数据库表结构对应，为持久化提供面向对象抽象。视图（View）由前端 Vue 渲染，后端不返回服务端模板页，从而形成“前端视图 + 后端 **API**”的清晰分工。

#### 4.1.3 Vue、Spring Boot、MySQL 与 Redis 协同及数据流

系统技术栈可概括为 **Vue 3**（表现层）、**Spring Boot**（应用服务层）、**MySQL**（业务数据持久化）与 **Redis**（热点数据缓存）的协同：**Vue** 侧通过 **Axios** 发起请求，经 **RESTful** 约定以 **JSON** 交换数据；请求进入后端 **Controller** 后转交 **Service** 处理业务，**Service** 通过 **Repository** 访问 **MySQL** 完成增删改查，并在适当时机读写的 **Redis** 以减轻数据库压力、加速排行榜与进度等高频访问。认证方面采用无状态 **JWT**：登录成功后服务端签发令牌，客户端在 `Authorization: Bearer <token>` 中携带，服务端校验签名与有效期后解析身份与角色，无需服务端集中存储会话，利于水平扩展。听书等功能由后端调用**百度语音合成**等外部 **HTTP API** 完成。典型调用链为：用户操作 → **Vue** 组件 → **RESTful** 请求 → **Controller** → **Service** → **Repository** / **Redis** → **MySQL**，响应再沿相反路径返回并由前端驱动视图更新；前端组件与响应式数据可对应 **MVVM** 中 View 与 ViewModel 的结合。接口以资源为中心，使用 **GET/POST/PUT/DELETE** 表达查询、创建、更新与删除语义，例如 `GET /api/novels/{id}` 获取小说详情、`POST /api/auth/login` 登录、`PUT /api/user/preferences` 更新阅读偏好。本系统未引入 **RPC**（如 gRPC、Dubbo），主要考虑浏览器生态对 **HTTP/JSON** 支持成熟、**RESTful** 风格易于文档化与联调。

### 4.5 接口设计概要

**读者端接口**（前缀如 /api，部分需登录）：

- **认证**：POST /auth/login、POST /auth/register，返回 JWT 及用户信息。
- **首页**：GET /novels/home，返回特色推荐、强推榜、新书榜、热点榜、首页推荐、完本推荐、新书精选、热门小说、文库列表等聚合数据。
- **小说**：GET /novels/{id} 详情；GET /novels/search 搜索；GET /novels/rankings 排行榜；GET /novels/completed-chart 完本榜单；GET /novels/browse 筛选；POST /novels/{id}/flower 送鲜花；POST /novels/{id}/favorite 加入书架（需登录）。
- **章节**：GET /chapters/novel/{novelId} 章节列表；GET /chapters/{id} 章节内容。
- **用户**：GET/PUT /user/profile 资料；GET/PUT /user/preferences 阅读偏好。
- **书架**：GET /bookshelf 列表；DELETE /bookshelf/{novelId} 移出书架。
- **分类**：GET /publishers、GET /tags 供前端下拉与筛选使用。

**管理端接口**（前缀如 /api/admin，均需管理员身份）：

- **管理端认证**：POST /admin/auth/login，请求体含管理员账号与密码，返回管理员 JWT（Token 中携带角色 ADMIN 或单独管理端 Token）。
- **用户管理**：GET /admin/users 用户列表（支持分页、关键词、状态筛选）；GET /admin/users/feedback 用户反馈列表；PUT /admin/users/{id}/status 禁用/启用账号；PUT /admin/users/feedback/{id} 处理反馈（回复、关闭等）。
- **小说管理**：GET /admin/novels 小说列表（支持分页与关键词）；POST/PUT /admin/novels 新增/编辑小说；PUT /admin/novels/{id}/shelf 上架/下架（若已实现）；GET /admin/chapters/novel/{novelId}、PUT /admin/chapters/{id}、POST /admin/chapters 章节列表与更新、新增章节。
- **分类管理**：GET/POST/PUT/DELETE /admin/publishers 文库的列表与增删改；GET/POST/PUT/DELETE /admin/tags 标签的列表与增删改；PUT /admin/novels/{id}/tags 维护某本小说的标签关联。
- **评论管理**：GET /admin/comments 评论列表（支持待审核/已通过/已拒绝、分页）；PUT /admin/comments/{id}/audit 审核通过/拒绝；DELETE /admin/comments/{id} 删除违规评论。
- **数据统计**：GET /admin/stats/reading 用户阅读数据（如按日/周/月汇总阅读量、活跃用户）；GET /admin/stats/novels 小说热度数据（阅读量、收藏、鲜花等排行）；GET /admin/stats/report 运营分析报表（可选导出）。
- **系统设置**：GET/PUT /admin/settings 系统参数配置；GET /admin/settings/server 服务器状态（可选）；GET /admin/logs 操作日志列表（支持时间、模块、操作人筛选与导出）。

读者端与管理端受保护接口均在请求头中携带 Authorization: Bearer <token>；管理端接口由 Spring Security 校验 Token 中的管理员角色，非管理员访问 /api/admin/** 返回 403。

### 4.6 安全设计

系统安全设计主要包括以下几方面。（1）认证与鉴权：采用无状态 JWT 认证。读者端登录（/auth/login）与管理端登录（/admin/auth/login）可共用同一 UserDetailsService 或分别使用读者账号表与管理员账号表；登录成功后签发 JWT，Token 中包含用户标识、角色（如 USER、ADMIN）与过期时间。请求时由 JwtAuthenticationFilter 解析并校验签名与过期时间，将用户信息与角色写入 SecurityContext。读者端受保护接口要求已认证用户；管理端接口（/api/admin/**）要求已认证且角色为 ADMIN，非管理员访问返回 403。（2）密码安全：读者与管理员密码均使用 BCrypt 算法单向加密后存储，不保存明文或可逆加密结果。（3）接口权限：读者端公开接口（如 /auth/login、/auth/register、/novels/**、/chapters/** 等）允许匿名访问；加入书架、书架列表与删除、用户资料与偏好等需登录后访问，否则返回 401。管理端所有接口均需管理员 Token，与读者端权限隔离，防止普通用户越权执行审核、上下架、用户禁用、系统设置等操作。（4）跨域与 CSRF：前后端分离场景下，通过 CORS 配置允许指定前端源（读者端与管理端可配置不同源或同一源不同路径）访问；本系统采用无状态 JWT，未使用基于 Cookie 的会话，因此禁用 CSRF 以简化配置，实际生产环境可结合网关与 HTTPS 进一步加固。（5）管理端操作审计：对管理员的关键操作（用户禁用、小说上下架、评论删除、系统参数修改等）写入 operation_log，便于事后追溯与安全审计。

## 第 7 章 总结与展望

### 7.1 总结

本系统设计并实现了一个**基于 Spring Boot 的轻小说在线阅读与听书平台**。在搭建过程中，系统选用 **Spring Boot** 作为后端开发框架，整合 Spring Security、Spring Data JPA 等组件完成业务与持久化；前端采用 **Vue.js**（Vue 3、Vite、Vue Router、Axios）构建单页应用，通过 **RESTful API** 与后端服务进行数据交互，前后端职责清晰，有利于系统扩展与维护，并提升了页面响应与开发迭代效率。在数据存储方面，系统使用 **MySQL** 持久化用户、小说、章节与评论等核心业务数据；相较部分重型或专用数据库方案，MySQL 在性能、可靠性与生态成熟度上表现均衡，部署与运维成本适中，能够较好满足本系统的业务规模与毕设实践需求。系统实现了**读者端**首页多模块推荐、小说浏览与筛选、全文搜索、用户注册登录、个人书架、阅读进度与阅读偏好、章节阅读，以及集成语音合成接口的听书能力；同时提供**管理端**用户管理、小说上架与章节维护、评论审核、标签与小说—标签关联等后台能力，为轻小说读者与管理员提供一体化的在线阅读与内容治理环境。通过本项目的开发与联调测试，验证了基于 **Spring Boot、Vue.js、MySQL**（并结合 **Redis** 缓存热点数据、**JWT** 完成鉴权）构建该类 Web 阅读平台的**可行性与实用性**。

系统功能已较为完善，**仍有进一步提升空间**，例如可引入更精细的个性化推荐以改善发现体验，或在安全审计、压力测试与自动化回归等方面持续加强；具体改进方向将在下一节作进一步说明。

### 7.2 不足与展望

当前系统存在以下不足：（1）管理端已实现并完成测试核心模块（用户管理、小说上架、评论审核、标签维护与小说-标签关联），但小说管理“更新”、数据统计与系统设置等页面当前仍以占位或未接入持久化接口为主；（2）首页推荐、排行榜等模块以配置驱动为主，缺乏基于用户行为的个性化推荐；（3）听书功能仅支持当前章节合成与播放，未提供语速、音色选择及断点续播；（4）阅读进度与偏好的多端同步依赖现有接口，在高并发或跨设备场景下的一致性保障有待加强；（5）未开展压力测试与自动化测试，性能与回归验证主要依赖手工测试。后续改进与计划包括：进一步完善管理端剩余模块并补充自动化回归测试；引入基于用户书架与阅读行为的简单推荐算法，丰富首页与“猜你喜欢”等模块；增强听书功能（语速、音色、断点续播）；结合 Redis 或消息机制优化阅读进度与偏好的多端同步；补充 JMeter 压力测试与 Selenium/Cypress 或接口自动化测试；视需求增加评论、书评、评分等社区功能及敏感词与安全防护。后续将根据实际使用反馈，在性能、安全与功能上持续优化与扩展。
